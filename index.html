HTML

Hypertext Markup Language (HTML) is one of the three cornerstone languages used in web development: HTML, CSS, and JS. HTML is used to create the structure of a webpage. It is made up of elements which work together to provide content and the basic framework for all pages on the web.

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user
  • section elements are semantic elements that define a section in a document that contains thematic content
  • The footer appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation

Introduction to Github

Git is a version control system that lets you manage and keep track of each version of your project. To access Git, we use the command line.

GitHub is a cloud-based hosting service that allows us to keep track of our projects inside repositories that are stored remotely. Using GitHub not only allows us to store our code, but also to easily collaborate with others.

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git Add -A: Add the change to be commited
  • git commit -m "": commit the chang to be pushed, m stands for message
  • git pull origin main: pulled the changes from the main branch
  • git push: pushed the change to github

Steps to Create a new branch

  1. You should be in main, if not switch to main branch git checkout main
  2. Pull down the latest main branch git pull origin main
  3. Checkout main branch and create a new branch Git checkout -b feature/name of the branch
  4. Verify you are in the new branch git status
  5. Open the project in VS Code if it is not open Code .

    6. After making changes the follwing steps will be for pushing the branch to GitHub
  6. to see the changes run git status
  7. to add the change to a commit git add -A
  8. to commit the change git commit -m "type your message here"
  9. before you commit you should pull the changes from the main to avoid conflicts git pull origin main
  10. to push the changes you made to your branch git push origin feature/name-of-your-branch
  11. Go to the GitHub repo in the browser, where the main branch is displayed by default. Select the branch feature/starter-code from the dropdown.
  12. Click the "Pull Requests" tab at the top, as shown in the following image
  13. Once on the pull requests page, find and click the "New pull request" button
  14. Now we need to compare the two branches that we want to merge. The base branch that we want to merge into should come first. In our case, this is the main branch. We want to merge our feature/your-name-branch, so that is our compare branch.
  15. Once on the pull request form, give your PR a title and a short description of the changes
  16. Next, select the "Create Pull Request" button to make your PR
  17. In the workplace and when working collaboratively as a team, once a PR has been created, we would request that one or two other team members review the code before it is merged into the base branch.
  18. if your PR reviwed and approved It is time to merge our new feature into our main codebase! Click the "Merge pull request" button and confirm
  19. Let's make sure that we close our HTML issue on GitHub since we have merged in the feature associated with it. Navigate to the "Issues" tab on GitHub
